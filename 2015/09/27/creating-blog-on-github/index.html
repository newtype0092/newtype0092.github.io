用Hexo在github上搭建静态博客

  • hexo blog

最近想搭一个blog写点东西，总结自己学到的东西。

先后考察过多种方案：博客园、oschina、WordPress、手撸Python、PHP…最后还是觉得这种静态博客最符合要求。

  • 为什么用静态博客?
    搭建简单，适合没点web技能树的程序员，省去各种乱七八糟的配置，让人专心与写作，简单的才是最好的。

  • 为什么在Github上？
    不用每天看国内社区的各种广告，也不用自己费心折腾VPS，而且还是免费的。经常上上Github看看自己惨不忍睹的Contributions，能鞭策自己多写两行代码…

  • 为什么用Hexo？
    官方推荐的是jekyll，看了一些教程后发现Hexo确实配置比较简单，而且速度更快。

Github配置

原理

Github是一个代码托管平台，为了方便开发者为自己或项目建立一个主页，官方提供了一个服务叫Pages，简单的说就是一个特殊的仓库，你将自己做的主页按要求的格式push上去，就可以通过固定的域名访问，但是主页必须是静态的。

Hexo就是一个将本地写的blog转成静态网页的工具，将转换后的网页文件push上去，通过分配的域名访问，就相当于是自己的blog了。
